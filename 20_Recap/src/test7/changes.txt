Can i have non static block at class level , what is it used for ?
ANS: Yes, you absolutely can! It is called an Instance Initializer Block (or simply a "Non-Static Block").
What is it? It is a block of code inside curly braces { ... } written directly inside the class (not inside a method).

What is it used for?
Shared Code: It runs for every object you create, regardless of which constructor you use. It is perfect for code that must happen for every instance (like assigning a unique ID or incrementing a counter) so you don't have to copy-paste it into every single constructor.
+1

Execution Order
Parent Constructor (if any)
Instance Block (Non-static block) //This runs BEFORE the constructor body, for every new object.
Child Constructor
